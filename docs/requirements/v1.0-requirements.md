# v1.0 需求文档

## 项目背景

统一 Rust 数据逻辑层，向下适配各平台 Native API，向上提供跨平台接口。UI 层（Flutter / 原生）无需再处理平台特性差异。

**典型使用场景**：数据库路径、缓存目录、设备信息等平台相关能力，由 Rust 侧统一提供。

## 目标平台

| 平台 | 系统 API | 桥接方式 |
|------|----------|----------|
| Android | NDK (C/C++) | FFI |
| iOS | UIKit/Foundation (ObjC/C) | FFI |
| HarmonyOS | NDK (C/C++) | FFI |
| Windows | Win32 API (C) | FFI |
| Linux | POSIX (C) | FFI |
| macOS | Cocoa (ObjC/C) | FFI |

## 功能规划

### v1.0 — 基础能力（优先实现）

| 功能 | 说明 |
|------|------|
| 平台信息 | 操作系统类型、版本、设备型号、CPU 架构 |
| 文件路径 | 应用数据目录、缓存目录、文档目录、临时目录 |
| 屏幕信息 | 分辨率、DPI、屏幕方向 |

### v1.1 — 存储相关

| 功能 | 说明 |
|------|------|
| KV 存储 | SharedPreferences / UserDefaults / Registry |
| 数据库路径 | SQLite 默认存储位置 |

### v2.0 — 系统能力

| 功能 | 说明 |
|------|------|
| 网络状态 | 在线/离线、WiFi/蜂窝 |
| 本地通知 | 系统通知推送 |
| 剪贴板 | 读写剪贴板 |
| 应用生命周期 | 前后台切换回调 |

### 暂不实现（Rust 生态已有方案）

| 功能 | 现有方案 |

> 这些能力虽不需适配，但使用时可参考本文档确认平台兼容性。

## 接口设计建议

### 同步 + 异步双接口

```rust
// 同步接口 — 简单场景
pub fn cache_dir() -> Result<PathBuf>;

// 异步接口 — 需要 I/O 等待的场景
pub async fn cache_dir_async() -> Result<PathBuf>;
```

**建议**：
- 纯数据获取（路径、设备信息）→ 默认同步
- 涉及 I/O 操作（文件读写、网络）→ 提供异步版本
- 内部实现可统一，异步版本包装同步调用即可

### 返回值规范

```rust
// 统一使用 Result，不 panic
pub fn platform_info() -> Result<PlatformInfo>;

// 自定义错误类型
pub enum PlatformError {
    NotSupported,      // 平台不支持
    PermissionDenied,  // 权限不足
    SystemError(i32),  // 系统错误码
}
```

## 平台差异处理策略

### 原则

| 情况 | 处理方式 |
|------|----------|
| 行为一致 | 统一接口，内部各自实现 |
| 行为有差异 | 提供统一接口 + 平台特有扩展 |
| 某平台不支持 | 返回 `Err(PlatformError::NotSupported)` |

### 示例

```rust
// 统一接口
pub fn cache_dir() -> Result<PathBuf>;

// 平台特有扩展（可选）
#[cfg(target_os = "android")]
pub fn external_cache_dir() -> Result<PathBuf>;  // Android 独有
```

### 路径规范

| 平台 | 应用数据目录 | 缓存目录 |
|------|-------------|----------|
| Android | `/data/data/<pkg>/files` | `/data/data/<pkg>/cache` |
| iOS | `~/Library/Application Support` | `~/Library/Caches` |
| Windows | `%APPDATA%\<pkg>` | `%LOCALAPPDATA%\<pkg>\Cache` |
| Linux | `~/.local/share/<pkg>` | `~/.cache/<pkg>` |
| macOS | `~/Library/Application Support/<pkg>` | `~/Library/Caches/<pkg>` |
